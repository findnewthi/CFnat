## 功能特性

- **智能优选**: 自动从 IP 池中筛选低延迟、低丢包的优质 IP
- **粘滞负载均衡**: 粘滞槽位 + Power of Two Choices，最多 5 个槽位同时工作，每个槽位有固定粘滞时长（10s, 15s, 20s...），避免频繁切换 IP
- **呼吸式扩张/收缩**: 流量多时自动扩张，流量少时自动收缩，动态调整槽位数量
- **动态调整**: 自动剔除劣质 IP，备选队列自动补位
- **Colo 过滤**: 支持按数据中心筛选 IP
- **双协议支持**: 自动识别 TLS/HTTP 流量，转发到对应端口

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-addr` | 本地监听地址 | `127.6.6.6:6` |
| `-colo` | 数据中心过滤，多个用逗号分隔 | 无 |
| `-dl` | 延迟上限（毫秒） | `300` |
| `-tlr` | 丢包率上限 | `0.1` |
| `-http` | 测速地址 | `http://cp.cloudflare.com/cdn-cgi/trace` |
| `-ips` | 目标负载 IP 数量 | `10` |
| `-n` | 测速并发数 | `128` |
| `-tp` | TLS 流量转发端口 | `443` |
| `-p` | HTTP 流量转发端口 | `80` |
| `-f` | IP 文件路径 | `ip.txt` |

## 示例

```bash
# 基本使用
./cfnat -f ip.txt

# 指定数据中心
./cfnat -f ip.txt -colo HKG,LAX,SJC

# 自定义参数
./cfnat -f ip.txt -ips 20 -dl 500 -n 256 -addr 0.0.0.0:443
```

## 工作原理

- 程序维护主队列和备选队列两个 IP 池
- 测速协程从 IP 池并发测速筛选优质 IP，每个 IP 发送 4 次 HEAD 请求计算平均延迟，支持按数据中心（Colo）过滤
- 通过延迟阈值的 IP 优先填充主队列，满后填充备选队列
- 用户请求时，使用粘滞槽位机制（四步流程）：
  - **Shrink（收缩）**：清理 15 秒没有访问的槽位
  - **Rotate（轮转）**：槽位到期后自动更换 IP，优先从主队列槽位外选择，没有则从备选队列选择
  - **Expand（扩张）**：双路径扩张（时间保底 + 连接数快路径），最多 5 个槽位，粘滞时长递增（10s, 15s, 20s, 25s, 30s）
  - **Route（路由）**：P2C 算法在槽位间负载均衡，选择连接数更少的槽位
- 核心特点：
  - **粘滞时长递增**：不同槽位的粘滞时长不同（10s, 15s, 20s, 25s, 30s），避免同时到期
  - **呼吸式扩张/收缩**：流量多时自动扩张（每 10 秒或连接数高时），流量少时自动收缩（15 秒无访问）
  - **双层缓冲**：主队列槽位外是第一层缓冲，备选队列是第二层缓冲
  - **空间分散，时间粘滞**：多个槽位同时工作（空间），每个槽位有固定粘滞时长（时间）
- 主队列 IP 通过用户请求被动检测，每次请求记录延迟和丢包，保留 20 次样本，超阈值则剔除并由备选补位
- 备选队列 IP 每 25 秒主动测速检测，同样超阈值剔除
- 整个系统形成 IP 池 → 测速筛选 → 备选队列 → 主队列 → 粘滞槽位（四步流程）→ 用户请求的流水线，剔除后自动补位
